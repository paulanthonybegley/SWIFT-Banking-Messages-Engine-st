SWIFT Message Documentation

Complete guide to SWIFT banking message formats and field specifications.

MT940 - Customer Statement Message

MT940 is used to send customer statement messages that list debits and credits to a customer's account.

Required Fields

Field Description Format Example
:20: Transaction Reference 16x REFERENCE123
:25: Account Identification 35x 123456789
:28C: Statement Number/Sequence 5n[/5n] 12345/1
:60F: Opening Balance 1!a6!n3!n15d C160101EUR1000,00
:62F: Closing Balance 1!a6!n3!n15d C160101EUR1100,00

Optional Fields

Field Description Format
:61: Statement Line Various
:64: Closing Available Balance 1!a6!n3!n15d
:86: Information to Account Owner 6*65x

MT942 - Interim Transaction Report

MT942 provides interim transaction information between full statement periods.

MT942 uses the same field structure as MT940 but is used for interim reporting.

MT101 - General Financial Institution Transfer

MT101 is used for direct financial institution transfers and payment orders.

Key Fields

Field Description Format
:20: Transaction Reference 16x
:32A: Original Date/Amount 6!n4!n1!a15d
:50K: Ordering Customer 6*35x

MT103 - Single Customer Credit Transfer

MT103 is a SWIFT payment message used for cross-border/international wire transfers.

Mandatory Fields

Field Description Format
:20: Transaction Reference 16x
:23B: Bank Operation Code 4!c (CRED, SPAY, etc)
:32A: Value Date/Currency/Amount 6!n3!a15d
:50K: Ordering Customer 4*35x
:59: Beneficiary Customer 4*35x
:71A: Details of Charges 3!a (OUR, BEN, SHA)

MT104 - Direct Debit and Request for Debit Transfer

MT104 is used for direct debit collections between financial institutions, supporting Straight-Through Processing (STP).

Sequence Structure

Sequence A: General Information (Mandatory)
Sequence B: Individual Transactions (Mandatory, repetitive)
Sequence C: Settlement Details (Mandatory for direct debits)
Field Description Sequence
:20: Transaction Reference Number A
:30G: Requested Value Date A
:21: Transaction Reference Number B
:32B: Currency/Amount B
:50K: Ordering Customer (Creditor) B
:19: Total Amount C
:30: Requested Execution Date C
